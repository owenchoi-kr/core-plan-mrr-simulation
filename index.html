Core Plan $30K MRR: 9월까지 가능한가?

2026-03-15 · GTM 시뮬레이션

전체 MRR 목표 $35K 중 기존 딥링크 플랜 $5K를 빼면, Core Plan이 만들어야 할 MRR은 $30K다. 현재 Core Plan MRR은 $0.

이를 위해 과금 고객 ~210개사(이탈 보충 포함)를 6개월 안에 만들어야 한다. 개발자 도구 벤치마크(OpenView 2023)로 역산하면 월 트래픽 ~11,700이 필요하고, 현재 US 트래픽(~5.8K/월)의 약 2배다.

1. 과금 고객 한 명이 월 얼마를 내나?

Core Plan은 인스톨당 $0.05를 과금한다. 월 1,250개(연 15K)까지는 무료이고, 그 이상부터 돈을 낸다.

고객의 UA(광고) 지출 규모에 따라 인스톨이 결정되고, 그게 곧 과금액이 된다.

2. 210개 과금 고객을 만들려면, 몇 명을 데려와야 하나?

데려오는 유저 전부가 돈을 내는 건 아니다. 개발자 도구 벤치마크(OpenView/boldstart 2023)에서 Signup→Paid median은 5%지만, Core Plan은 SDK 설치가 필수라 3%를 base로 잡는다.

3. 그러면 트래픽을 얼마나 부어야 하나?

개발자 도구 전용 벤치마크(OpenView/boldstart 2023, 수백 개사)를 사용한다:

• Visit → Signup: 10% (개발자 도구 median)

• Signup → Paid: 3% (dev tool median 5%에서 SDK 마찰 감안, 일반 SaaS 10%의 1/3)

Core Plan은 SDK 설치→채널 연결이 필수라 개발자 도구 중에서도 마찰이 높다. 5% median이 아닌 3%를 base로 잡는다.

계산: 트래픽 = 210 ÷ (Visit→Signup × Signup→Paid)

4. 결론

$30K Core Plan MRR, 9월까지 가능한가?

조건부로 가능하다. 하지만 두 가지를 동시에 해결해야 한다:

1
Signup→Paid 전환율 올리기 (SDK 설치가 관건)
SDK 필수 제품 base 3%에서 시작, 5%(dev tool median)까지 올리는 게 목표. SDK 설치 마찰을 줄이는 게 핵심이다 — 가입 후 SDK 설치까지의 드랍이 전환율을 지배한다. 3%면 월 11,700, 5%면 월 7,000으로 줄어든다.
2
트래픽 파이프라인 구축
Core Plan 전용 가입자는 현재 ~0이다. (기존 US 트래픽 5.8K/월은 Growth Plan 시절.) base 3% 기준 월 11,700이면 현재의 2배. 퍼널을 5%까지 올려도 1.2배는 필요하다.
