Core Plan $30K MRR: 9월까지 가능한가?

2026-03-15 · GTM 시뮬레이션

전체 MRR 목표 $35K 중 기존 딥링크 플랜 $5K를 빼면, Core Plan이 만들어야 할 MRR은 $30K다. 현재 Core Plan MRR은 $0.

이를 위해 과금 고객 ~210개사(이탈 보충 포함)를 6개월 안에 만들어야 한다. 개발자 도구 벤치마크(OpenView 2023)로 역산하면 월 트래픽 7,000~11,700이 필요하고, 현재 US 트래픽(~5.8K/월)의 1.2~2.0배 수준이다.

1. 과금 고객 한 명이 월 얼마를 내나?

Core Plan은 인스톨당 $0.05를 과금한다. 월 1,250개(연 15K)까지는 무료이고, 그 이상부터 돈을 낸다.

고객의 UA(광고) 지출 규모에 따라 인스톨이 결정되고, 그게 곧 과금액이 된다.

2. 210개 과금 고객을 만들려면, 몇 명을 데려와야 하나?

데려오는 유저 전부가 돈을 내는 건 아니다. 개발자 도구 벤치마크(OpenView/boldstart 2023, 수백 개사)에 따르면 Signup→Paid는 5%다. 일반 SaaS(10%)의 절반 수준이고, SDK 설치가 필수인 제품은 이보다 더 낮을 수 있다.

3. 그러면 트래픽을 얼마나 부어야 하나?

개발자 도구 전용 벤치마크(OpenView/boldstart 2023, 수백 개사)를 사용한다:

• Visit → Signup: 10% (개발자 도구 median)

• Signup → Paid: 5% (개발자 도구 median, 일반 SaaS 10%의 절반)

Core Plan은 SDK 설치가 필수라 개발자 도구 중에서도 마찰이 높은 편이다. 2-3% 시나리오도 봐야 한다.

4. 결론

$30K Core Plan MRR, 9월까지 가능한가?

조건부로 가능하다. 하지만 두 가지를 동시에 해결해야 한다:

1
Signup→Paid 전환율 올리기 (SDK 설치가 관건)
개발자 도구 median 5%가 목표. SDK 설치 마찰을 줄이는 게 핵심이다 — 가입 후 SDK 설치까지의 드랍이 전환율을 지배한다. 5%면 월 7,000, 3%면 월 11,700 트래픽이 필요하다.
2
트래픽 파이프라인 구축
Core Plan 전용 가입자는 현재 ~0이다. (기존 US 트래픽 5.8K/월은 Growth Plan 시절.) 전환율 5% 기준 월 7,000이면 현재 트래픽의 1.2배. 3%이면 2배가 필요하다.
